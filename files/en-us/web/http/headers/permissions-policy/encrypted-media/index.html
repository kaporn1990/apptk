The Permissions-Policy header was previously named Feature-Policy. As of June 2021, the {{domxref('FeaturePolicy')}} API as well as some linked-to articles and demos still use the older name.

The HTTP {{HTTPHeader("Permissions-Policy")}} header encrypted-media directive controls whether the current document is allowed to use the Encrypted Media Extensions API (EME). When this policy is enabled, the {{jsxref("Promise")}} returned by {{domxref("Navigator.requestMediaKeySystemAccess","Navigator.requestMediaKeySystemAccess()")}} will reject with a {{domxref("DOMException")}}.

Syntax

Permissions-Policy: encrypted-media <allowlist>;
<allowlist>
A list of origins for which the feature is allowed. See Permissions-Policy.

Default policy

Default allow list for encrypted-media is 'self'.
